DatePicker

DatePicker er en plugin.

DatePicker er ikke med pr. default.

Filerne js/calendar.min.js og css/calendar.min.js eksisterer, men skal tilføjes til projektet (filerne kan også tilføjes uden .min til debug).

DatePicker har følgende begrænsning pr. default:

  • Range: 01-01-1900 – 01-01-2100
  • Ingen fremhævet dage (weekend er normalt fremhævet)
  • Callback til dato validering (beskrevet nedunder)

Default datepicker tildeles med klassen skts-date-picker. Feltet skal have en id.

Datoen skrives som dd-mm-åååå
<label for="datePickerTest" id="datePickerTestLabel">Dato</label>
<input type="text" class="form-control skts-date-picker" id="datePickerTest" name="datePickerTest" value=""  data-show-type="dkDate" data-validation-event="blur"/>
<div class="skts-error-text" role="alert">Datoen skrives som dd-mm-åååå</div>

Der er tre alternativ datepicker:

  • js-date-picker-fixed-year: hvor brugeren kun kan vælge indenfor det år som er sat i input type="hidden" id="sktsCurrentYear".
  • js-date-picker-fixed-start-year: hvor brugeren kun kan vælge fra det år som er sat i input type="hidden" id="sktsCurrentYear" (slut datoen er ikke sat).
  • js-date-picker-with-range: hvor brugeren kun kan vælge fra og til datoer. Værdierne sættes i atributterne data-calendar-start-date og data-calendar-end-date. Kalenderen kan sættes til at starte i en bestemt måned med atributten data-calendar-cursor-start-date.

Default datepicker og de alternative datepickere kan IKKE bruges sammen. Dvs det er ENTEN klassen .js-date-picker-fixed-year, .js-date-picker-fixed-start-year, .js-date-picker-with-range eller .skts-date-picker som skal sættes på feltet.

Følgende js funktioner kan bruges til oprettelse/tilintetgørelse:

  • datePickcer.init() - opretter alle datePickers med de ovennævnte klasser
    • Denne funktion køres onload når man inkulderer css/calendar.min.js filen
  • datePicker.create(inputElement, startDate, endDate, cursorStart) - opretter en datePicker på et inputfelt
    • @param {Object} input som objekt
    • @param {string} dato med formattet: dd-mm-åååå (f.eks 01-01-2016)
    • @param {string} dato med formattet: dd-mm-åååå (f.eks 01-01-2016)
    • @param {string} dato med formattet: dd-mm-åååå (f.eks 01-01-2016) - optional
  • datePicker.destroy(datePickerID) - tilintetgøre en datePicker baseret på feltets id
    • @param {string} id på input feltet, uden "#"
  • datePicker.destroyAll() - tilintetgøre alle datePickers med de ovennævnte klasser

Chosen select

Chosen select er ikke med pr. default.

Filerne js/chosen.min.js og css/chosen.min.css eksisterer, men skal tilføjes til projektet (js filerne kan også tilføjes uden .min til debug).

Chosen select er en plugin som udvider standard select boksen.

Alle select bokse kan omdannes til en chosen select boks ved brug af klassen .chosen-select

Feltet er påkrævet

Temaer

En stylesheet/tema kan testes på denne guide ved at tilføj querystrengen: css og placering/navn på css filen (uden .min eller .css):

  • Default
  • theme-skts-erhverv-2015

Temaer er ren css og bruges til ændring/justering af SKATs standard design.

Temaet indeholder både nye klasser og overstyret bootstrap klasser/variabler.

Et tema er en less fil som bygges seperat af gulp og bliver lagt i css mappen .

Less filen navngives theme-skts-TEMANAVN.less (hvor TEMANAVN beskriver temaets formål, f.eks theme-skts-monbile-moms.less).
